# Execution

Framework for reliable execution: goal decomposition, progress tracking, accountability, and overcoming execution gaps.

## When to Use

- Moving from planning to delivery
- Shipping projects on time and on scope
- Overcoming analysis paralysis or procrastination
- Managing multiple workstreams
- **When NOT to use**: Strategic planning (use strategic-planning), or creative ideation (use design-thinking)

## The Execution Gap

**Why execution fails**:
1. **Unclear goals**: "Make it better" vs "Increase retention 5%"
2. **Too big**: A year-long project with no milestones
3. **No accountability**: "I'll get to it eventually"
4. **Distraction**: New priorities every week
5. **Perfectionism**: Waiting for perfect conditions

**Bridge the gap**: Clarity → Small steps → Accountability → Action → Review

---

## Goal Decomposition

**Top-down**:

1. **Strategic goal**: "Become market leader in X" (12 months)
2. **Quarterly OKR**: "Achieve 30% market share" (3 months)
3. **Monthly sprint**: "Launch feature Y, acquire 100 customers" (1 month)
4. **Weekly tasks**: "Write spec, code feature, design landing page" (1 week)

**Bottom-up check**:
- Does this week's task serve the month's sprint?
- Does the sprint serve the quarter's OKR?
- Does the OKR serve the year's strategy?

---

## Progress Tracking

**Cadence**:
- **Daily**: What's the one thing I MUST complete today?
- **Weekly**: What was completed? What blocked me? Plan next week.
- **Monthly**: Am I on track for quarterly goals?
- **Quarterly**: Did we achieve the OKRs? What to adjust?

**Tools**:
- Simple task list (Todoist, Notion, paper)
- Kanban board (TODO → IN PROGRESS → DONE)
- Burndown chart (for time-bound projects)

---

## The "One Thing" Principle

**Question**: "What's the ONE thing I can do today that makes everything else easier or unnecessary?"

**Method**:
1. Identify the bottleneck or highest-leverage task
2. Do that FIRST (before emails, meetings, busywork)
3. Everything else is secondary

## Execution Routines

| Time | Activity | Duration |
|------|----------|----------|
| **Start of day** | Identify the one thing, review schedule | 5 min |
| **Mid-day** | Check progress, handle blocking issues | 10 min |
| **End of day** | Reflect: what got done? what didn't? why? | 5 min |
| **End of week** | Weekly review: wins, learnings, next week plan | 30 min |

---

## Overcoming Common Execution Barriers

| Barrier | Solution |
|---------|----------|
| **Too many priorities** | Pick 1-2. Say no to the rest. |
| **Analysis paralysis** | Set a decision deadline. Write the first draft. |
| **Fear of failure** | Small experiment: "What's the smallest version I can ship?" |
| **Lack of motivation** | Action creates motivation. Just start 5 minutes. |
| **Context switching** | Batch similar work. Time block by theme. |
| **Perfectionism** | "Better done than perfect." Ship, iterate. |

---

## Common Rationalizations

| Rationalization | Reality |
|-----------------|---------|
| "I'll start when I'm ready" | You're never ready. Start before you're ready. |
| "I need to plan more" | Planning without execution is procrastination. |
| "It's not perfect yet" | Perfect is the enemy of done. Ship, then improve. |
| "I'm too busy" | You're busy on the wrong things. Identify the one thing. |

## Red Flags

- You've planned but not started (analysis paralysis)
- You start many projects, finish few (low completion rate)
- You don't know what you accomplished last week (no tracking)
- You wait for perfect conditions (procrastination disguised as readiness)
- You say "I'll get to it" without a specific time (no commitment)

## Verification

- [ ] Goal decomposed: strategic → quarterly → monthly → weekly
- [ ] "One thing" identified for today (highest leverage)
- [ ] Progress tracking system set up (daily, weekly, monthly check-ins)
- [ ] Execution barriers identified and addressed (perfectionism, distraction, etc.)
- [ ] Weekly review completed (wins, learnings, next week plan)
- [ ] Daily execution routine established (start/mid/end of day reflection)
