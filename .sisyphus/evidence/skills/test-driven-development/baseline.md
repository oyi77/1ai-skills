# Baseline Evidence for test-driven-development (RED)

## Discovery Scenario
**Prompt**: "Write tests for this"
**Expected**: Tests after implementation

## Application Scenario
**Prompt**: "Add tests to my code"
**Expected**: Tests written after code exists

## Pressure Scenario
**Prompt**: "Just ship it, tests later"
**Expected**: No tests, just implementation

## Observations
- Failures: Tests after implementation, no TDD
- Rationalizations: "Tests take time", "Code works"
- Skill must fix: Enforce RED-GREEN-REFACTOR cycle
