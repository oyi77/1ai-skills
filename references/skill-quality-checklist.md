# Skill Quality Checklist

Use this checklist when creating or reviewing any SKILL.md.

## Frontmatter (Required)

- [ ] `name` matches directory name (kebab-case)
- [ ] `description` starts with action verb
- [ ] `description` is 50-200 characters
- [ ] `domain` matches category directory
- [ ] `tags` has 3-5 relevant tags (lowercase kebab-case)

## Structure (Required)

- [ ] `# Title` тАФ clear, descriptive
- [ ] `## When to Use` тАФ 3+ trigger conditions
- [ ] `## When NOT to Use` тАФ 2+ contraindications
- [ ] `## Overview` тАФ 2-3 sentence summary
- [ ] `## Workflow` тАФ numbered steps with bold labels
- [ ] `## Verification` тАФ checklist of success criteria

## Content Quality

- [ ] Imperative language ("Run X" not "You should run X")
- [ ] Specific commands and tools named
- [ ] Concrete examples over abstract theory
- [ ] No template placeholder text
- [ ] No generic filler ("This section covers...")
- [ ] Under 500 lines total
- [ ] Every step is actionable
- [ ] Every verification is checkable

## Consistency

- [ ] No duplicate description with other skills
- [ ] No broken `/skills/` internal links
- [ ] Tags are relevant and not redundant
- [ ] Domain matches actual content
