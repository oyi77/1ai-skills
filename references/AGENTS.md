# references/

## Purpose
Static reference checklists agents can load for domain best practices. **Not full skills** — these are passive lookup tables, not actionable workflows. For actionable skills with workflows + personas, see the category dirs (`development/`, `marketing/`, etc.).

## Files

| File | Domain |
|---|---|
| `code-review-checklist.md` | Code review criteria |
| `marketing-checklist.md` | Marketing campaign QA |
| `seo-checklist.md` | SEO audit items |
| `trading-checklist.md` | Trade execution checklist |

## For AI Agents
- Treat as **read-only reference material**.
- Pair with a category skill (e.g. `references/code-review-checklist.md` + `development/code-reviewer/SKILL.md`) — the skill drives behavior, the checklist enforces completeness.
