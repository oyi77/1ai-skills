# Developer Marketing & Launch Strategy Reference

Developers don't respond to traditional marketing. They evaluate through code,
documentation, and community reputation. This reference covers distribution
channels specific to technical/SaaS products.

---

## Developer Marketing Principles

1. **Show, don't tell.** Demos > decks. Code > claims. Docs > brochures.
2. **Earn trust through contribution.** Blog posts, open-source, answers on Stack Overflow.
3. **Remove friction.** Self-serve trial, no sales call required, clear docs.
4. **Speak their language.** Technical accuracy matters. BS detectors are calibrated.
5. **Respect their time.** Short tutorials, scannable docs, copy-paste examples.

## Product Hunt Launch Playbook (2025-2026)

### Pre-Launch (4-6 Weeks Before)

**Week 1-2: Build the Foundation**
- Create Product Hunt maker profile (establish before launch)
- Build a "Coming Soon" page on Product Hunt
- Start building an email list (target: 100-500 before launch)
- Begin teasing on Twitter/X and LinkedIn

**Week 3-4: Prepare Assets**
- Product Hunt listing: 5 images, 1 video (90 seconds max)
- Tagline: under 60 chars, clear value prop
- First comment: write a genuine founder story (why you built this)
- Prepare "ask" message to send supporters on launch day

**Week 5-6: Build Launch Team**
- Contact 50-100 people who will support on launch day
- Send early access to 20-30 people for testimonials
- Schedule social posts for launch day
- Prepare email blast to your list

### Launch Day

**Timing:** Launch at 12:01 AM PST (Product Hunt resets daily)

**First 2 Hours (Critical):**
```
□ Post goes live at 12:01 AM PST
□ Send "we're live" email to supporters
□ Post on Twitter/X with link
□ Share in relevant Slack/Discord communities
□ First comment: your founder story
□ Respond to EVERY comment within 15 minutes
```

**Hours 2-12:**
```
□ Keep engaging in comments (be genuine, not just "thanks!")
□ Share updates on social media every 2-3 hours
□ Send reminder to supporters who haven't voted
□ Post in relevant Reddit communities (value-first, not spammy)
□ Reach out to relevant newsletter/media contacts
```

**Hours 12-24:**
```
□ Continue engaging (momentum matters)
□ Share traction updates ("Top 5 of the day!")
□ Thank supporters publicly
□ Monitor and respond to all comments
```

### Post-Launch (Next 7 Days)
- Follow up with everyone who commented
- Write a "lessons learned" post about your launch
- Reach out to interested users for demos/onboarding
- Submit to other directories while you have momentum

### Key Metric
Only 10% of launches get featured. Success factors:
- Quality of product (actually good, not just well-marketed)
- Engagement velocity in first 4 hours
- Genuine community interaction (not fake upvotes)
- Quality of assets and story

## GitHub as Marketing Channel

### README as Landing Page
Your GitHub README IS a landing page for developers.

**README Structure:**
```markdown
# {Product Name}
{One-line description}

{Demo GIF or screenshot — VISUAL FIRST}

## Why {Product}?
{3 bullet points: problem → solution → result}

## Quick Start
{Copy-paste installation in 3 commands or less}

## Features
{Feature list with brief descriptions}

## Examples
{2-3 practical code examples they can run immediately}

## Documentation
{Link to full docs}

## Community
{Discord/Slack link, Twitter, contributing guide}

## License
{License type}
```

### GitHub Growth Tactics
- Star your own related repos (build graph connections)
- Contribute to popular repos in your space (PR, issues, discussions)
- Post "Show HN" style updates when you release features
- Add GitHub Topics tags (SEO for GitHub search)
- Submit to awesome-* lists in your category
- Make the repo actually useful as standalone (not just marketing)

### GitHub Stars as Social Proof
```
100 stars    → Credibility signal for early adopters
500 stars    → Featured in GitHub Trending potential
1,000 stars  → Category authority
5,000+ stars → Significant distribution channel
```

## Hacker News Strategy

### What Works on HN
- "Show HN:" posts about genuinely interesting technical work
- Deep technical blog posts (architecture, scaling, algorithms)
- Honest failure/learning posts from building
- Open-source tool announcements
- Data-driven industry analysis

### What Gets Killed on HN
- Marketing speak ("revolutionize," "disrupt," "game-changing")
- Landing page submissions (no substance)
- Self-promotional comments
- Astroturfing (fake accounts, coordinated upvoting)

### HN Submission Tips
- Title: factual, specific, no clickbait
- Submit between 8-10 AM EST (peak engagement)
- Be active in comments (founders engaging = good)
- Answer technical questions thoroughly
- Accept criticism gracefully (HN is harsh but fair)
- Comment on your own post with context/story

## Documentation-Led Growth

### Why Docs Drive Revenue
- Developers evaluate products through documentation
- Good docs reduce support costs (self-serve answers)
- Docs pages rank for long-tail developer queries (SEO value)
- API docs with copy-paste examples reduce time-to-value

### Documentation Structure
```
Getting Started         → 5-minute quickstart
Installation            → Every platform covered
Authentication          → API keys, OAuth, etc.
Core Concepts           → How the product thinks
API Reference           → Every endpoint documented
Code Examples           → Copy-paste examples in 3+ languages
Tutorials               → Step-by-step for common use cases
FAQ / Troubleshooting   → Common issues with solutions
Changelog               → What's new (keeps content fresh for SEO)
```

## Partnerships & Distribution

### Integration Marketplace Marketing
- Build integrations with complementary tools (Zapier, Make, n8n)
- List on integration marketplaces (each listing = backlink + discovery)
- Co-market with integration partners (joint blog posts, case studies)
- The integration page itself is SEO content: "{Your Product} + {Partner}" keywords

### AppSumo Launch (Revenue Burst)
- 1.4M+ tech-savvy buyers on AppSumo
- Typical SaaS deal: lifetime access at steep discount
- Use for: initial revenue burst, user acquisition, feedback
- Risk: attracting deal-hunters who never convert to full price
- Best strategy: AppSumo for v1 → collect feedback → raise prices for v2

### SaaS Directory Submissions (One-Time, Compound Returns)
Submit to 20+ directories for SEO backlinks and discovery:
```
General:        G2, Capterra, GetApp, Product Hunt, AlternativeTo
AI tools:       There's An AI For That, FutureTools, AI Tool Directory
Dev tools:      StackShare, LibHunt, awesome-* lists
SaaS specific:  SaaSHub, SaaSWorthy, AppSumo Marketplace
Open source:    Open Source Alternative, FOSS directory
Regional:       ToolFinder.id (Indonesia), etc.
```

## Webinar / Live Demo Funnel

### The Solo Founder Webinar Framework

**Why:** Webinar attendees convert at 8-12% (vs 2-3% for cold traffic).
**How often:** Monthly or bi-weekly for consistency.
**Format:** 20 minutes teaching + 5 minutes product demo + 5 minutes Q&A.

### Evergreen Webinar Funnel
```
Traffic → Registration Page → Thank You + Calendar Invite
                                    ↓
                            Reminder email (1 day before)
                            Reminder email (1 hour before)
                                    ↓
                              Live/Recorded Webinar
                                    ↓
                            Follow-up email (1 hour after)
                            Offer email (24 hours after)
                            Last chance (48 hours after)
                                    ↓
                              Not converted → Nurture sequence
                              Converted → Onboarding
```

**Record once, replay forever.** After running 2-3 live webinars and refining,
turn the best version into an evergreen automated webinar.
