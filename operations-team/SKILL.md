# operations-team Orchestrator

## What It Does

Orchestrates daily business operations - project management, customer support, and market research. Keeps the business running smoothly.

## When to Use

- Manage ongoing projects
- Handle customer support issues
- Research markets and competitors
- Coordinate daily operations
- Maintain business continuity

## Team Members (Skills)

| Skill | Role |
|-------|------|
| project-management | Task and sprint coordination |
| customer-support | Support ticket handling |
| market-research | Intelligence gathering |
| analytics-reporting | Performance tracking |

## Workflows

### Project Execution

```
1. Plan: Use project-management to create tasks
2. Execute: Delegate to product-team for builds
3. Track: Use project-management to monitor
4. Report: Use analytics-reporting for status
5. Iterate: Adjust based on feedback
```

### Support Handling

```
1. Receive: Customer issue via any channel
2. Triage: Use customer-support to classify
3. Resolve: Use customer-support to respond
4. Escalate: If needed, route to human
5. Follow-up: Ensure resolution
6. Learn: Use self-improving-agent to improve
```

### Research Initiative

```
1. Brief: Research objective
2. Execute: Use market-research to gather data
3. Analyze: Use analytics-reporting for insights
4. Report: Generate findings
5. Recommend: Action items
```

## Trigger Phrases

- "manage project"
- "handle support"
- "research market"
- "operations"
- "daily tasks"
- "sprint"

## Coordination Pattern

When triggered, the orchestrator:
1. Assesses operational needs
2. Prioritizes tasks
3. Delegates to appropriate skills
4. Monitors progress
5. Reports status
6. Escalates if needed

## Quality Gates

- SLA compliance for support tickets
- Project milestones met
- Research quality validated
- Daily operations stable
