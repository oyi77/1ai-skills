## 🚀 Scripts

### `bot_extractor.py` — Extract Bot Architecture

```bash
# Full extraction (recommended)
python3 scripts/bot_extractor.py @targetbot --session alwayscuanbos

# Quick mode (depth 2 only, faster)
python3 scripts/bot_extractor.py @targetbot --quick

# Save to file
python3 scripts/bot_extractor.py @targetbot --output architecture.json

# Extract + generate clone blueprint
python3 scripts/bot_extractor.py @targetbot --blueprint --output full_arch.json

# Quiet (no verbose output)
python3 scripts/bot_extractor.py @targetbot --quiet --output arch.json
```

**What it extracts:**
- All `/commands` and responses
- Every inline button + `callback_data`
- Full menu tree via BFS (all depths)
- Input flow states: `WAITING_IMAGE` | `WAITING_TEXT` | `WAITING_FILE` | `WAITING_VIDEO`
- User data exposed in responses (email, phone, Telegram ID)
- URL buttons → external service hints
- Response timing → sync vs async AI backend
- Tech stack fingerprinting

**Output JSON:**
```json
{
  "bot": "@targetbot",
  "extracted_at": "...",
  "user_data": {"email": "...", "telegram_id": "..."},
  "commands": {"/start": {"text": "...", "buttons": [...]}},
  "menus": {
    "__root__": {"trigger": "/start", "buttons": [...]},
    "Menu A": {"trigger": "cb_a", "depth": 1, "input_state": null},
    "Menu A > Sub": {"trigger": "cb_b", "depth": 2, "input_state": "WAITING_IMAGE"}
  },
  "input_flows": [{"trigger": "cb_x", "state": "WAITING_IMAGE", "path": [...]}],
  "button_data_map": {"cb_a": "Button Label"},
  "tech_hints": {"framework": "aiogram", "max_processing_min": 5}
}
```

**Known limitations:**
- Cannot access admin-only menus (needs admin account as session)
- URL buttons (not callback) are detected but not followed
- Rate limiting: adds 2-3s delay between clicks (required by Telegram)
- Deep bots (>5 levels) may take 5+ minutes

### `bot_cloner.py` — Generate Bot Code + UX Audit

```bash
# Generate working aiogram 3.x bot
python3 scripts/bot_cloner.py architecture.json --output ./my_bot/

# Specify framework
python3 scripts/bot_cloner.py architecture.json --output ./my_bot/ --framework aiogram

# UX audit only (no code generation)
python3 scripts/bot_cloner.py architecture.json --improve
```

**Generated files:**
```
my_bot/
├── bot.py                  ← Entry point (aiogram Dispatcher)
├── config.py               ← BOT_TOKEN placeholder
├── states.py               ← FSM states (BotStates)
├── requirements.txt        ← aiogram, aiohttp, python-dotenv
├── handlers/
│   ├── commands.py         ← /start, /help handlers
│   ├── callbacks.py        ← All inline button handlers
│   └── messages.py         ← Input state handlers (WAITING_*)
└── keyboards/
    └── menus.py            ← All InlineKeyboardMarkup definitions
```

**After generating:**
```bash
cd my_bot/
pip install -r requirements.txt
# Edit config.py → set BOT_TOKEN
# Edit handlers/messages.py → implement AI/processing logic
python3 bot.py
```

**UX Audit criteria:**
| Check | Penalty |
|-------|---------|
| Dead buttons (no callback_data) | -20 pts |
| Missing /help | -10 pts |
| Missing /cancel | -10 pts |
| Menu depth > 3 | -10 pts |
| Menus without back button | -5 pts |
| Vague input prompts | -3 pts each |

## 🔄 Full Workflow

```
1. IDENTIFY bot → get @username

2. EXTRACT
   python3 scripts/bot_extractor.py @targetbot --output arch.json

3. REVIEW architecture
   cat arch.json | python3 -m json.tool | head -100

4. AUDIT UX
   python3 scripts/bot_cloner.py arch.json --improve

5. GENERATE CLONE
   python3 scripts/bot_cloner.py arch.json --output ./clone/

6. CUSTOMIZE
   └── Edit handlers/messages.py → add your AI/API calls
   └── Edit config.py → add BOT_TOKEN

7. DEPLOY
   python3 clone/bot.py
```
