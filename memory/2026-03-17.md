## 8. Lessons Learned

**Issue: Bot Not Responding to Commands**
- Cause: Service was running in production mode (NODE_ENV=production)
- Production mode requires WEBHOOK_URL for Telegram webhooks
- Development mode uses polling (no webhook needed)
- Solution: Set `NODE_ENV=development` in systemd service

**Issue: GeminiGen Placeholder URLs**
- Cause: Service was using mock implementation (example.com/video.mp4)
- Solution: Integrated real GeminiGen API with proper authentication
- Test confirmed: Real video URLs generated successfully

**Issue: Service Dependency Management**
- Problem: Bot needs PostgreSQL & Redis, but they weren't guaranteed to be running
- Solution: Added `ExecStartPre` directives to start Docker containers before bot
- Result: Service now manages its own dependencies automatically

**Best Practice: Systemd Service Design**
- Use `PartOf=` to tie child service lifecycle to parent
- Use `After=` to ensure proper startup order
- Use `Restart=always` for resilience
- Use `ExecStartPre=-` (with dash) to ignore failures in dependency startup
- Always set proper `Environment=` variables in service file
